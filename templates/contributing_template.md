# Contributing to [Project Name]

Thank you for deciding to contribute! Please review the guidelines below.

---

## 🚀 How to Contribute

### 1. Issue Reporting
- Search existing open and closed issues first.
- If not found, open an issue with a clear title and reproduction steps.

### 2. Pull Requests
1. Fork the repository and create your branch from `main`.
2. Follow the language-specific styling guidelines.
3. Write clean, modular, and self-documenting code.
4. Add unit or integration tests for new features.
5. Create a descriptive pull request outlining modifications.

---

## 🎨 Code Style

- **Python**: Adhere to PEP 8 standards. Use `black` or `ruff` for formatting.
- **JavaScript**: Follow ES6+ conventions. Use `prettier` for style consistency.
- **Git**: Write clean commit messages in imperative style (e.g., `feat: add core verification loop`).
